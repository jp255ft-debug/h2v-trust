# .clinerules for H2V-Trust

## Stack Tecnológica
- **Backend:** Python 3.11, FastAPI, SQLAlchemy, TimescaleDB, Alembic
- **Frontend:** Next.js 14 (App Router), TypeScript, Tailwind CSS, shadcn/ui
- **Blockchain:** Solidity, Hardhat, Web3.py (Polygon/Hardhat)
- **Infra:** Docker, Docker Compose (dev & prod), Nginx, WSL2

## Comandos Essenciais
- `make dev-start` — Iniciar ambiente dev
- `make dev-check` — Verificar saúde do ambiente
- `make dev-reset` — Reset completo do ambiente
- **NUNCA** executar dois comandos Docker em simultâneo

## Diretórios Chave
- `backend/api/routes/` — Endpoints da API
- `backend/services/` — Lógica de negócio
- `backend/core/` — Regras de compliance e certificação
- `frontend/app/` — Páginas Next.js 14 (App Router)
- `frontend/src/lib/api.ts` — Cliente HTTP do frontend
- `contracts/contracts/` — Smart contracts Solidity

## Security
### Sensitive Files
DO NOT read or modify:
- `.env` files
- `.env.production`
- Any file containing API keys, tokens, or private keys

### Security Practices
- Never commit sensitive files
- Use environment variables for secrets
- Keep credentials out of logs and terminal output

## Architecture Decisions
- All new backend features must be added to `backend/api/routes/` with a corresponding service in `backend/services/`.
- All new frontend pages must be added under `frontend/app/` following Next.js 14 App Router conventions.
- Any changes to the database schema require a migration script in the `alembic/` directory.

## Maintenance
- Update `CHANGELOG.md` when adding features.
- Keep `README.md` in sync with new capabilities.
